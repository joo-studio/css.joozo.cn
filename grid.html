We are using a standard 12 column fluid responsive grid system.The grid helps you layout your page in an ordered, easy fashion

Container

The container class is not strictly part of the grid but is important in laying out content. It allows you to center your page content. The .ct-container class is set to ~70% of the window width. It helps you center and contain your page content. We use the container to contain our body content.

Demo

Try the button below to see what the page looks like without containers. If you want the page full screen, just put .ct-container nice change .ct-container-fuild

Turn off Containers

To add a container just put your content inside a <div> tag with a .ct-container class. Here's an example of how your page might be set up.

                    
    <body>
      <div class="ct-container">
        <!-- Page Content goes here -->
      </div>
    </body>

Offsets

To offset, simply add .ct-sm-offset-2 to the class where .ct-sm signifies the screen class-prefix (sm = small, md = medium, lg = large) and the number after is the number of columns you want to offset by.

This div is 12-columns wide on all screen sizes
6-columns (offset-by-6)

    <div class="ct-row">
      <div class="ct-sm-12 grid-example"><span class="ct-my-lg">This div is 12-columns wide on all screen sizes</span></div>
      <div class="ct-sm-6 ct-sm-offset-6 grid-example"><span class="ct-my-lg">6-columns (offset-by-6)</span></div>
    </div>

Creating Layouts

Here we will show you how to create some commonly used layouts with our grid system. Hopefully these will get you more comfortable with laying out elements. To keep these demos simple, the ones here will not be responsive.

Section

The section class is used for simple top and bottom padding. Just add the .ct-section class to your div's containing large blocks of content.

Divider

Dividers are 1 pixel lines that help break up your content. Just add the .ct-divider to a div in between your content.

Adding Responsiveness

In the previous examples, we only defined the size for small screens using .ct-sm-12. This is fine if we want a fixed layout since the rules propagate upwards. By just saying ct-sm-12, we are essentially saying "ct-sm-12 ct-md-12 ct-lg-12". But by explicitly defining the size we can make our website more responsive. The code "ct-sm-12 ct-md-12 ct-lg-12" you can also write this "ct-lg-12" or "ct-md-12" or "ct-sm-12 ct-lg-12"However, the most correct wording is such that "ct-sm-12" Because the Csstoolkits is Mobile first.

I am always full-width (ct-sm-12)
I am full-width on mobile (ct-sm-12 ct-md-6)

    <div class="ct-row">
      <div class="grid-example ct-sm-12"><span class="ct-my-lg">I am always full-width (ct-sm-12)</span></div>
      <div class="grid-example ct-sm-12 ct-md-6"><span class="ct-my-lg">I am full-width on mobile (ct-sm-12 ct-md-6)</span></div>
    </div>

Responsive Side Navigation Layout

In this example below, we take the same layout from above, but we make it responsive by defining how many columns the div should take up on each screen size. Try resizing your browser and watch the layout change below.


    <!-- Navbar goes here -->

    <!-- Page Layout here -->
    <div class="ct-row">

      <div class="ct-md-4 ct-lg-3"> <!-- Note that "ct-md-4 ct-lg-3" was added -->
        <!-- Grey navigation panel

              This content will be:
          3-columns-wide on large screens,
          4-columns-wide on medium screens,
          12-columns-wide on small screens  -->

      </div>

      <div class="ct-md-8 ct-lg-9"> <!-- Note that "ct-md-8 ct-lg-9" was added -->
        <!-- blue page content

              This content will be:
          9-columns-wide on large screens,
          8-columns-wide on medium screens,
          12-columns-wide on small screens  -->

      </div>

    </div>
